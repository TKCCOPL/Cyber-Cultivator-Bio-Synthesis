# 高级种子（Advanced Seed）模型推导与参数说明

## 【模型目标】
建立一个可验证的数学模型，描述玩家通过两种路径获得“高级种子”的概率与速率：

- 路径A：直接掉落（每次采集独立触发，概率为 p0）。
- 路径B：通过交配/晋级（多次尝试逐级晋升，达到等级 L 获得高级种子）。

模型需能够评估在大量并行设备（N 台）下的长期产率、首次到达时间分布，以及服务器负载风险，并给出缓解策略。

---

## 【基本假设】
1. 採集事件（harvest）为独立伯努利试验，成功获得高级种子的概率为 p0。每台采集装置的采集频率为 hps（harvest per second）。
2. 交配/晋级抽象为一维马尔可夫链，状态 0→1→...→L，只有向上转移。每次交配尝试独立产生一次“晋级成功”概率 μ（突变率）。达到等级 L 即视作产生一个高级种子。
3. 每次交配尝试耗时 Tb（秒），每台交配设备以速率 1/Tb 发起尝试。
4. 不考虑降级、复杂基因重组、多位点遗传效应（可作为扩展）。

---

## 【公式与推导】

参数符号表：

- p0: 直接掉落概率（per harvest）
- μ: 单次交配晋级概率（per attempt）
- L: 达到等级 L 视为高级种子
- Tb: 单次交配耗时（秒）
- hps: 单台采集装置每秒采集次数
- Nh: 采集装置数量
- Nb: 交配装置数量
- N = Nh + Nb

1) 交配路径的期望尝试次数：

  E_attempts = L / μ

 说明：每次晋级为几何分布，期望需要 1/μ 次成功晋级，累积 L 次成功期望为 L/μ 次尝试。

2) 交配路径的期望时间（单台设备）：

  E_time_breeding = E_attempts * Tb = (L / μ) * Tb

3) 直接掉落路径的期望时间（单台采集设备）：

  E_time_harvest = (1 / p0) / hps = 1 / (p0 * hps)

4) 并行产率（长期稳态近似）:

  - 采集产率（高级种子/秒）： R_h = Nh * hps * p0
  - 交配产率（高级种子/秒）： R_b = Nb * μ / (L * Tb)

  因此总体产率 λ ≈ R_h + R_b

推导说明：交配路径近似将 L 次成功视为 E_attempts 次尝试的集合，故每 E_attempts 次尝试会产生 1 个高级种子；每秒交配尝试次数为 Nb/Tb，因此产率近似为 (Nb/Tb)/(E_attempts) = Nb * μ / (L * Tb)。

5) 极端规模示例：Nb=10000, μ=0.02, L=5, Tb=30s → R_b ≈ 10000 * 0.02 / (5*30) = 1.3333 高级种子/秒。

---

## 【服务器负载评估与缓解】

问题核心：高并发不在于产率本身，而是每秒需要执行的事件数（ticks、扫描、网络同步）。事件率 E_events ≈ Nb/Tb + Nh*hps。

缓解策略（数学/工程）：

1. 批处理与均摊：每秒只处理一部分设备，或将昂贵计算拆分到多帧。
2. 聚合速率模型：在服务器端按 Nb 汇总计算晋级速率，而不是对每台设备单独仿真。即使用 λ_b = Nb * μ / (L * Tb) 的速率模型。
3. 边际收益递减（防止线性放缩）：

   - 简单衰减： μ_eff = μ / (1 + β * max(0, density - D0))
   - Logistic 抑制： μ_eff = μ / (1 + exp(α*(Nb - N_half)))

4. 限流与配额：对单玩家设备数量设置上限或随并发加收成本（如能量/消耗系数上升）。

## 【建议默认参数（用于仿真）】

- p0 = 0.0005
- μ = 0.02
- L = 5
- Tb = 30.0
- hps = 0.2

这些参数用于后续 Python 验证脚本，可按需调整。

---

## 【如何使用仿真脚本】

- 脚本位置：`simulations/advanced_seed_sim.py`。
- 运行示例：

```bash
pip install tqdm
python simulations/advanced_seed_sim.py --trials 50000 --Nh 9000 --Nb 1000
```

输出将包含平均首次到达时间（秒）、中位数以及估算的长期产率等统计量。
