# Cyber-Cultivator: Bio-Synthesis — 使用指南

## 模组简介

赛博农夫 (Cyber-Cultivator) 是一个中型 Minecraft Forge 模组，用精密的实验室设备取代传统农业，融合遗传育种算法与生物强化血清系统。你需要通过挖掘矿物、种植生化作物、培育基因、合成血清来强化自身能力。

**可选依赖：** Curios API（饰品）、KubeJS（配方 DSL 与玩法事件）、Patchouli `1.20.1-85-FORGE`（游戏内指导书）。未安装任何可选依赖时，机器、培养、血清和基因系统仍可正常使用。本模组兼容 JEI。

## 快速开始

安装 Patchouli 时，玩家首次进入世界会直接收到中英文《生物合成研究手册》。发放标记随玩家持久化，重登和死亡不会重复发放；旧世界后装 Patchouli 时，下次登录仍会获得。丢失后可用「书 + 硅碎片」无序合成补领。

### 第一步：挖掘基础矿物

进入地下矿洞，找到两种模组矿石。石头层生成普通变种，负高度的深板岩层还会生成对应的深板岩变种：

| 矿石 | 石镐挖掘 | 掉落物 | 用途 |
|------|---------|--------|------|
| 硅晶矿石 (Silicon Ore) | 需要石镐 | 粗硅晶矿 | 烧制为硅碎片 |
| 深板岩硅晶矿石 (Deepslate Silicon Ore) | 需要石镐 | 粗硅晶矿 | 烧制为硅碎片 |
| 稀土矿石 (Rare Earth Ore) | 需要石镐 | 粗稀土矿 | 烧制为稀土粉末 |
| 深板岩稀土矿石 (Deepslate Rare Earth Ore) | 需要石镐 | 粗稀土矿 | 烧制为稀土粉末 |

四种矿石均支持精准采集保留原方块，时运对普通掉落物数量生效；深板岩变种不会替换已有普通矿石的方块 ID。

粗硅晶矿和粗稀土矿需要先在熔炉或高炉中精炼，分别产出硅碎片和稀土粉末；高炉速度是熔炉的两倍。粗矿可在工作台中以 `3×3` 压制成「粗硅晶块」或「粗稀土块」，精炼材料则可压制成光滑的「硅晶块」或「稀土块」。四种储存块均可无序拆解回 9 份对应材料，并需要石镐或更高级的镐完整采集。

### 第二步：种植基础作物

在创造模式物品栏「赛博农夫」标签页中找到三种种子：

主标签提供的三种基础种子均为速度、产量、效价 `5/5/5`，原料、突触神经莓和血清在此只展示品质 5 代表。相邻的「赛博农夫：品质样本」标签按等级排列三种种子的均衡基因样本（`1/1/1` 至 `10/10/10`）、品质 `1-10` 的三种灌装原料，以及 Activity `1-15` 的突触神经莓和三种血清。种子的三个基因可以独立变化，因此此处不生成会达到每种 1000 个的全组合样本。

| 作物 | 收获物 | 用途 |
|------|--------|------|
| 纤维草 (Fiber Reed) | 植物纤维 | 莓合成配方输入 |
| 蛋白质豆 (Protein Soy) | 生化原液 | 莓合成/S-01 配方输入 · 培养槽 N 通道注入剂 |
| 酒精花 (Alcohol Bloom) | 工业乙醇 | 莓合成/S-03 配方输入 |

**生存来源：** 破坏草或高草丛有 `10%` 几率掉落纤维草种子。蛋白质豆与酒精花种子保留原有宝箱来源（每个匹配宝箱 `20%`），并新增学徒级村民交易：农民以 `3 绿宝石 → 2 蛋白质豆种子`，牧师以 `3 绿宝石 → 2 酒精花种子`，每项最多交易 8 次。

> 作物种植在耕地上，与原版小麦类似，但最终产物需在培养槽中精耕。
>
> 三种原料都是「双重身份」物品：既可作为血清灌装机的配方输入（参与 Activity 计算），也作为培养槽某条通道的注入剂（共享池消耗，与品质无关）。详见下方「生物培养槽」章节的「共享池说明」。

## 核心系统详解

### 大气冷凝器

放入玻璃瓶后自动运行，每 30 秒消耗 1 个玻璃瓶并产出 1 个「纯净水瓶」，库存上限 16 个。玻璃瓶耗尽或库存已满时冷凝暂停并保留当前进度；手动、Shift 点击或漏斗抽取也不会清空下一轮进度。旧存档已有的 17-32 瓶不会被截断；机器暂停生产，取至 16 瓶以下后自然恢复。

- **自动传输：** 将冷凝器放置在培养槽正上方，冷凝器会自动向下注入纯净度 +20（消耗 1 瓶纯净水）
- **漏斗兼容：** 顶部或侧面输入玻璃瓶，底部或侧面抽取纯净水瓶；输入与输出不会串槽
- **红石信号：** 有库存时输出红石信号 15
- **GUI：** 普通右键打开界面；玻璃瓶输入、散热鳍片、竖向冷凝柱、流向箭头和纯净水输出按横向链路分区，下方独立显示进度/剩余时间、库存与下游连接状态
- **运行方式：** 供瓶后自动生产；下方存在培养槽时自动注入纯净度，无需额外开关
- **快捷操作：** 潜行右键直接收取全部库存

### 生物培养槽

培养槽是模组的核心耕作方块。放入基因种子后，需要维持三项数值才能让作物生长：

| 数值 | 作用 | 注入方式 | 衰减速度 |
|------|------|---------|---------|
| 营养度 (N) | 低于阈值停止生长 | 生化原液 +25 | 每秒 -1 |
| 纯净度 (P) | 影响作物品质 | 纯净水瓶 +20 或冷凝器自动 | 每 2 秒 -1 |
| 数据信号 (D) | 高级作物必需 | 硅碎片 +15 | 每 3 秒 -1 |

> **共享池说明：** N/P/D 三项数值属于培养槽本身的共享池，**不属于种子或配方**。无论放入哪种种子、培养哪个配方，三项数值都会被同一台培养槽中的作物消耗。换种子不会清空或重置这三项数值。培养槽 GUI 上方实时显示三项数值；移除种子时，三项数值保留。

**营养度与「生化原液」的关系（重要）：**
- 「生化原液」既是培养槽的 **N 通道注入剂**（放入 N 槽消耗 +25），也是 **莓合成与 S-01 血清的配方输入**（直接进入血清灌装机）。
- 同一瓶「生化原液」用作 N 通道注入剂时不会保留品质信息；用作配方输入时其 `Concentration` 品质参与 Activity 计算。
- 「蛋白质豆」种植在培养槽中产出「生化原液」，其 `Concentration` 品质由蛋白质豆种子的 `Gene_Potency` 决定。
- 「植物纤维」「工业乙醇」同样是「双重身份」物品：前者是莓合成配方输入，后者同时是莓合成与 S-03 配方输入。两者在培养槽中不作为注入剂使用，但「硅碎片」既是 D 通道注入剂，也是单片镜合成材料。

**交互操作：**

| 操作 | 方法 |
|------|------|
| 打开界面 | 普通右键培养槽 |
| 放入种子 | 将基因种子放入种子槽 |
| 注入纯净度 | 将纯净水瓶放入 P 槽，自动消耗（不返还空瓶） |
| 注入营养度 | 将生化原液放入 N 槽，自动消耗并补充营养度 |
| 注入数据信号 | 将硅碎片放入 D 槽，自动消耗并补充数据信号 |
| 收取成熟资源 | 从界面右侧资源出口取出，或由底部漏斗抽取 |
| 取回种子 | 潜行 + 右键 |
| 查看状态 | GUI 显示 N/P/D 实时数值、生长进度、预计时间及当前阻塞原因 |

资源放入后不会瞬间注满：培养槽每 20 tick（1 秒）执行一次注入周期，每个周期在 N/P/D 三条通道中各最多消耗一份，直到对应储量满载。GUI 和容器仅保留同一行 5 个槽位（种子、N、P、D、成熟资源）；侧面漏斗只可输入三种培养资源，底部漏斗只抽取成熟资源。旧存档隐藏的空瓶会在首次打开时转入背包，满包则掉落；直接破坏机器也会掉落。

界面状态行会区分等待种子、缺少 N/P/D 中的具体通道、正常培养及 ETA、成熟后资源出口已满、产物就绪。N/P/D 数值直接显示在对应竖条上方，大生长条只表达实际进度，不播放额外装饰动画。

**生长条件：** N > 10 且 P > 10 且 D > 0 时，作物持续生长。进度满 200 后消耗额外营养和纯净度，将原料送入右侧资源出口并清空种子槽；出口无法容纳产物时保持满进度等待收取。

**连续速度公式：** `基因速率 = 0.5 + Speed/10×1.5`，`环境系数 = 0.65 + 0.35×(N+P+D)/300`。机器每 tick 以千分位累积两者乘积，满 1000 推进 1 点正式进度；Speed 1-10 因此每级都会加速。余量随区块保存，只在更换或取出种子时清零；GUI ETA 与实际生长共用同一速率。

> 提示：机器状态统一在培养槽 GUI 中查看；光谱单片镜只负责解析种子物品提示中的隐藏基因数据。

### 基因拼接机

将两颗基因种子放入拼接机后，机器会自动开始 5 秒拼接并产出新种子。加工期间连接条会播放物质汇流动画，右侧细进度条显示完成比例和剩余时间；移除任一输入会取消本次拼接并清零进度，重新补齐后自动从头开始。每颗种子携带三个基因属性（1-10）：

界面不会展开显示父本 A/B 的 S/Y/P，也没有开始按钮。预览只显示「子代代数·特殊突变率·最终双生率」和剪裁至基因上下限的普通 S/Y/P 范围，不再显示容易被误解为最终值的「平均中心值」。特殊突变可能超出普通范围；完成后改为显示实际基因、突变类型和数量。

- **Gene_Speed** — 生长速度
- **Gene_Yield** — 产量
- **Gene_Potency** — 效价（影响血清品质）

**拼接公式：**
```
普通基因 = floor((父本 + 母本) / 2) + 随机浮动(-2 ~ +2)
突变率 = min(5% + min(最高代数, 20)×0.5% + 最大基因差异×1%, 25%)
普通双生率 = min(10% + 最高代数×2%, 60%)
最终两颗概率 = 突变率 + (1-突变率)×普通双生率
```

每次固定消耗两颗亲本。普通结果产出 1-2 颗，特殊突变必定产出 2 颗，总上限为 2；同一次双生的两颗种子共享完全相同的基因和 NBT。

**操作方法：**

| 操作 | 方法 |
|------|------|
| 打开界面 | 普通右键拼接机 |
| 放入种子 | 将两颗基因种子放入 A/B 槽，第二颗进入后自动开始 |
| 查看进度 | 连接动画与细进度条显示 5 秒拼接进度 |
| 收取子代 | 普通点击、GUI Shift 快速移动或潜行右键 |
| 快捷取出 | 潜行右键优先取结果，否则取回最后一颗输入 |

拼接成功生成子代时会立即消耗两颗父本；如果拼接事件被取消，父本会保留。子代等待领取期间不能放入下一组父本。

> 策略提示：多次迭代育种可逐步提升基因数值，但变异范围 ±2 意味着每次都有退化风险。建议保留优质亲本。

### 血清灌装机

将突触神经莓加工为高级血清。3 个输入槽 + 1 个输出槽。成功饮用任意一种血清都会返还玻璃瓶：最后一支直接替换手持物，堆叠饮用则进入背包，满包时掉落。创造模式或被事件取消的饮用不产生空瓶。

**配方：**

| 配方 | 输入材料 | 输出 |
|------|---------|------|
| 莓合成 | 植物纤维 + 工业乙醇 + 生化原液 | 突触神经莓 |
| S-01 突触超频 | 莓 + 生化原液 + 玻璃瓶 | S-01 血清 |
| S-02 视觉强化 | 莓 + 稀土粉末 + 玻璃瓶 | S-02 血清 |
| S-03 代谢加速 | 莓 + 工业乙醇 + 玻璃瓶 | S-03 血清 |

- 加工时间：15 秒（300 tick）
- 漏斗兼容：顶部/侧面注入材料，底部抽取输出
- 交互：普通右键打开 GUI，放入任意顺序的 3 项材料后自动开始；单个材料包依次沿一条输入支路、上方总线和灌装入口移动，三路轮流播放，灌装进度条和输出箭头显示后续流程
- 信息：固定三行显示状态、产出/耗时和预计活性；输出受阻时会提示先收取产物，长配方名会自动截断
- 进度重置：GUI 不提供取消按钮；加工中改变或取出任一输入会自动取消旧进度且不消耗材料
- 快捷取出：潜行右键直接取出成品

## JEI 机器配方页面

安装可选 JEI 后，四类机器配方会直接使用对应机器 GUI 的工作区、真实槽位位置和加工动画，不再显示与游戏内界面脱节的独立配方卡。将鼠标悬停在槽位或说明区域可查看完整公式与资源规则。

| 页面 | 显示内容 |
|------|----------|
| 培养槽产出 | 种子、N/P/D 三种注入资源、实际成熟资源，以及默认基因与产量摘要 |
| 基因拼接 | 两颗父本、5 秒自动拼接、子代代数、特殊突变/双生概率与普通基因范围 |
| 血清灌装 | 当前 JSON/KubeJS 配方的三项输入、准确输出、15 秒加工时间，以及 Activity 计算或继承规则 |
| 大气冷凝 | 玻璃瓶输入、30 秒生产周期、纯净水输出、16 瓶库存上限与下游纯净度注入用途 |

JEI 只负责展示，不参与机器状态或配方选择；未安装 JEI 时四台机器及其 GUI 仍可正常运行。

## 血清系统

### S-01 突触超频血清

**合成方式：** 血清灌装机 = 突触神经莓 + 生化原液 + 玻璃瓶

**效果：**
- 攻击速度 +10%~45%（随 amplifier 增长，每级 +5%）
- 力量 I~VIII（随 amplifier 增长）
- 击退抗性 +10%~45%（随 amplifier 增长，每级 +5%）
- 基础持续时间：15 秒（再按 Activity 乘以 0.6~2.0），完整累加上限 90 秒

**副作用：** 药效结束或被提前清除后进入「神经过载」状态 — 弱化 + 挖掘疲劳，高等级追加凋零 I。

### S-02 视觉强化血清

**合成方式：** 血清灌装机 = 莓 + 稀土粉末 + 玻璃瓶

**效果：**
- 夜视效果
- 私有轮廓侦测范围 16-64 格（I~VIII 每级 +7 格），范围内所有生物显示仅饮用者可见的穿墙轮廓
- 扫描周期随等级从 60 tick 缩短至 32 tick
- 基础持续时间：20 秒（再按 Activity 乘以 0.6~2.0），完整累加上限 120 秒

**副作用：** 药效结束或被提前清除后进入「神经过载」状态 — 失明，IV 级起追加恶心，VII 级起追加缓慢 I。

### S-03 代谢加速血清

**合成方式：** 血清灌装机 = 莓 + 工业乙醇 + 玻璃瓶

**效果：**
- 食物值高于 6 时，每秒恢复 1~4.5 点生命值并增加额外饥饿消耗
- 移动速度 +5%~40%（随 amplifier 增长，每级 +5%）
- 跳跃提升 I~IV
- 基础持续时间：10 秒（再按 Activity 乘以 0.6~2.0），完整累加上限 60 秒

**副作用：** 药效结束或被提前清除后进入「神经过载」状态 — 缓慢 + 饥饿，V 级起追加中毒 I，VIII 级为中毒 II。

### 副作用缓解

「神经过载」根据血清来源施加不同副作用：
- **S-01 后遗症：** 弱化 + 挖掘疲劳，高等级追加凋零 I
- **S-02 后遗症：** 失明，高等级追加恶心和缓慢 I
- **S-03 后遗症：** 缓慢 + 饥饿，高等级追加中毒

可通过以下方式缓解：
- 佩戴「生命支持箱」饰品 — 以默认约 3 倍速度缩短神经过载持续时间
- 等待自然消退

普通牛奶可以提前结束血清主效果，但仍会触发对应神经过载；神经过载本身不接受牛奶清除。

## Curios 饰品系统

模组提供 3 件 Curios 饰品，需安装可选的 Curios API 才能佩戴和生效。单片镜可合成；腰带和生命支持箱仍通过指定战利品箱获取。

### 光谱单片镜 (Spectrum Monocle)

- **槽位：** 头部
- **功能：** 佩戴后查看基因种子物品提示，可解析 S/Y/P 基因值、代数、协同与突变信息；不再显示机器 HUD，机器状态请打开对应 GUI
- **合成：** 上排铁粒 + 玻璃板 + 铁粒，中排硅碎片 + 稀土粉末 + 硅碎片，下排中央铁粒
- **获取：** 仅通过合成获得，不再加入宝箱战利品

### 生化脉冲腰带 (Bio-Pulse Belt)

- **槽位：** 腰部
- **功能：** 每 5 秒自动扫描范围内培养槽，消耗背包材料注入数值
  - 营养度 < 阈值 → 消耗背包中生化原液 +25
  - 纯净度 < 阈值 → 消耗背包中纯净水瓶 +20
  - 数据信号 < 阈值 → 消耗背包中硅碎片 +15
- **获取：** 废弃矿井、要塞走廊/交叉口/图书馆、大型水下废墟战利品箱（20% 几率）
- **可配置：** 扫描范围（默认 3 格）、各项注入阈值

### 生命支持箱 (Life Support Pack)

- **槽位：** 背部
- **功能：**
  - 持续缓解神经过载副作用（默认额外消减 2 tick/tick，总计约 3 倍消退速度）
  - 低血量应急：HP ≤ 6 时自动消耗背包中生化原液，回复 4HP + 急迫效果（60 秒冷却）
- **获取：** 废弃矿井、地牢、掠夺者前哨站战利品箱（20% 几率）

突触神经莓是制造所有高级血清的核心中间产物。

**合成：** 血清灌装机 = 植物纤维 + 工业乙醇 + 生化原液

同时也是食物（回复 2 饱食度），但主要用途是作为血清配方的原料。

## 品质链路系统

原料和血清具有品质属性，从种子基因继承并通过加工链传递。

### 原料品质 NBT

培养槽中的作物成熟时，产出的原料会根据种子的 **Gene_Potency** 写入品质 NBT：

| 原料 | NBT 标签 | 来源 |
|------|---------|------|
| 植物纤维 | Potency | 纤维草种子的 Potency 基因 |
| 工业乙醇 | Purity | 酒精花种子的 Potency 基因 |
| 生化原液 | Concentration | 蛋白质豆种子的 Potency 基因 |

> 注：Gene_Purity 已在 v1.1.2 重命名为 Gene_Synergy（协同基因），避免与乙醇品质 Purity 混淆。

### 突触活性 (Synaptic Activity)

血清灌装机在合成莓或血清时，根据三种原料的品质计算**突触活性**：

```
Activity = round(Potency × 0.25 + Purity × 0.375 + Concentration × 0.375)
最终 Activity = clamp(基础 Activity + floor(Gene_Synergy / 2), 1, 15)
```

同一批原料中取最高的 `Gene_Synergy`（0-10）作为协同加成；加工成血清时直接继承神经莓的 Activity。活性值越高，血清效果越强。无品质 NBT 的原料默认品质为 5，缺失 Synergy 时加成为 0。

### 血清效果缩放

血清的实际效果按 Activity 缩放：
- **持续时间倍率：** `0.5 + Activity × 0.1`（Activity 1 = 0.6x，Activity 10 = 1.5x，Activity 15 = 2.0x）
- **起始等级：** Activity ≥ 8 时起步 II 级，否则起步 I 级
- **突破加成：** Activity > 10 后，每超出 2 点再提升 1 级首次效果，仍受 VIII 级总上限约束

## 血清叠加升级机制

多次饮用同种血清可叠加效果：

| 机制 | 规则 |
|------|------|
| 等级叠加 | 每次饮用 amplifier +1，上限 VIII 级（amplifier 7） |
| 持续时间累加 | 每瓶完整加入本瓶持续时间；S-01/S-02/S-03 分别上限 90/120/60 秒 |
| 高品质起步 | Activity ≥ 8 的血清首次饮用即为 II 级 |
| 副作用增强 | 等级越高，药效结束或被提前清除后的来源专属神经过载越严重，持续时间为 8~22 秒 |

三种血清可以同时存在并分别叠加。达到 VIII 级且持续时间也达到对应上限后，继续饮用会被阻止且不会消耗血清。

## 红石链路与比较器

四台机器均支持红石控制与比较器输出，便于接入红石自动化系统。

### 红石控制模式

每台机器 GUI 右上角有 `RS` 按钮，单击循环切换三种模式：

| 模式 | 按钮标签 | 行为 |
|------|----------|------|
| IGNORE | `RS:忽略` | 忽略红石信号，始终运行（默认） |
| HIGH | `RS:高电平` | 有红石信号时运行，无信号时冻结进度 |
| LOW | `RS:低电平` | 无红石信号时运行，有信号时冻结进度 |

- 加工中失去许可时**冻结进度**，恢复许可后继续，不清零、不消耗输入。
- 旧存档机器默认 `IGNORE`，不崩溃。
- 红石作用范围：冷凝器控制产水（已有纯净水仍可向下传输）；培养槽控制生长与成熟（资源注入和自然衰减继续）；拼接机控制拼接进度；灌装机控制配方选取与进度。

### 比较器信号

对四台机器背面放置比较器，输出三段语义信号：

| 信号 | 含义 |
|------|------|
| 0 | 待机：无产物且未加工 |
| 1-14 | 加工中：`ceil(progress × 14 / maxProgress)` |
| 15 | 产物就绪：主产物槽存在可抽取物品 |

主产物定义：冷凝器=纯净水；培养槽=成熟培养产物；拼接机=子代种子；灌装机=灌装成品。

> **兼容性变更**：当前版本将培养槽比较器语义从“营养度/7”统一为状态信号（0/1-14/15）。

### 公开 API

```java
// 查询机器红石控制状态
MachineControlInfo info = CyberCultivatorAPI.getMachineControlInfo(level, pos);
// info.mode() / info.powered() / info.processingAllowed() / info.comparatorSignal()

// 修改红石模式（仅服务端主线程）
boolean ok = CyberCultivatorAPI.setMachineRedstoneMode(level, pos, RedstoneControlMode.HIGH);
```

## 自动化产线布局

```
[大气冷凝器]        ← 输入玻璃瓶，每 30 秒产出纯净水
      ↓ 自动注入纯净度
[生物培养槽]        ← 放入种子，腰带自动注入营养/信号
      ↓ 作物成熟后取出原料
[血清灌装机]        ← 原料合成莓 → 莓+辅料+瓶 → 血清
      ↓ 输出
[突触神经莓 / S-01 / S-02 / S-03 血清]
```

冷凝器可从顶部/侧面漏斗输入玻璃瓶，并由底部/侧面漏斗抽取纯净水；灌装机顶部/侧面注入材料、底部抽取输出。

### 分面槽位矩阵

四台机器通过 Forge `IItemHandler` 能力暴露分面槽位，支持原版漏斗、Create、Mekanism、AE2、沉浸工程等遵循 Forge 能力的物流系统：

| 机器 | 顶部 | 水平面（北/南/东/西） | 底部 |
|------|------|----------------------|------|
| 大气冷凝器 | 玻璃瓶只入 | 玻璃瓶只入、纯净水只出 | 纯净水只出 |
| 生物培养槽 | 基因种子只入 | 营养/纯净水/信号只入 | 成熟产物只出 |
| 基因拼接机 | 两个亲本槽只入 | 两个亲本槽只入 | 子代只出 |
| 血清灌装机 | 三个材料槽只入 | 三个材料槽只入 | 成品只出 |

- 能力插入会自动规范化种子（数量限制为 1、补齐基因 NBT）。
- 模拟插入/抽取（`simulate=true`）不修改库存、进度或配方缓存。
- 输出槽拒绝外部插入，输入槽拒绝外部抽取。

> **兼容性变更**：当前版本开放培养槽顶部基因种子自动输入，并将基因拼接机改造为 `WorldlyContainer`，支持原版漏斗路径。

### 跨模组材料标签

本模组向外发布以下 Forge 约定标签，便于跨模组材料识别：

- `forge:silicon` → `silicon_shard`
- `forge:ores/silicon`、`forge:ores/rare_earth`
- `forge:raw_materials/silicon`、`forge:raw_materials/rare_earth`
- `forge:dusts/rare_earth`
- `forge:storage_blocks/silicon`、`forge:storage_blocks/raw_silicon`
- `forge:storage_blocks/rare_earth`、`forge:storage_blocks/raw_rare_earth`

本模组机器输入通过 `cybercultivator:` 语义标签受控扩展，整合包可通过数据包扩展输入物品，但不会自动把所有外部硅或普通种子视为合法机器材料。

### 跨模组兼容

机器遵循 Forge `IItemHandler` 分面能力和 `forge:*` 材料标签。Create、Mekanism、AE2、沉浸工程、Thermal、Pipez 等能够访问标准物品能力的物流系统预计可按上表接入，但尚未逐个进行运行时验证；具体方向识别、过滤器和主动抽取行为仍以对应模组实现为准。

## KubeJS 可选兼容

安装 KubeJS `2001.6.5-build.16` 至 `2001.6.5-build.26` 及其 Rhino、Architectury 前置后，可以在 `kubejs/server_scripts/` 中添加培养槽和灌装机配方，并监听四类玩法事件。KubeJS、Patchouli、JEI、Curios 均为可选依赖，发布 JAR 不捆绑它们。

### 专用配方 DSL

```javascript
ServerEvents.recipes(event => {
    event.recipes.cybercultivator.serum_bottling(
        'cybercultivator:synaptic_serum_s01',
        [
            'cybercultivator:synaptic_neural_berry',
            'cybercultivator:biochemical_solution',
            'minecraft:glass_bottle'
        ]
    )
        .processingTime(200)
        .inheritActivity(true)
        .inheritMutation(false)
        .priority(20)
        .id('kubejs:fast_s01')

    event.recipes.cybercultivator.incubator_output(
        'minecraft:apple',
        '#minecraft:saplings'
    )
        .countFormula('1 + yield / 2')
        .qualityTag('Potency')
        .defaultGenes({ speed: 5, yield: 5, potency: 5 })
        .cropName('实验树苗')
        .priority(10)
        .id('kubejs:experimental_sapling')
})
```

培养槽 `seed` 同时接受单个物品和物品标签。灌装配方必须有 1 至 3 项输入，`processingTime` 至少为 1 tick。所有可选字段省略时沿用原生默认值，`priority` 默认为 `0`。

### 原始 JSON / `event.custom()`

专用 DSL 最终生成原生数据包配方。也可以直接使用 `event.custom()`：

```javascript
ServerEvents.recipes(event => {
    event.custom({
        type: 'cybercultivator:serum_bottling',
        ingredients: [
            { item: 'cybercultivator:synaptic_neural_berry' },
            { item: 'minecraft:glass_bottle' }
        ],
        result: { item: 'cybercultivator:synaptic_serum_s02' },
        processing_time: 300,
        inherit_activity: true,
        inherit_mutation: false,
        priority: 5
    }).id('kubejs:custom_s02')

    event.custom({
        type: 'cybercultivator:incubator_output',
        seed: { tag: 'minecraft:saplings' },
        output: { item: 'minecraft:apple' },
        count_formula: '1 + yield / 2',
        quality_tag: 'Potency',
        default_genes: { speed: 5, yield: 5, potency: 5 },
        crop_name: '实验树苗',
        priority: 5
    }).id('kubejs:custom_sapling')
})
```

删除和替换遵循 KubeJS 的标准配方规则：

```javascript
ServerEvents.recipes(event => {
    event.remove({ id: 'cybercultivator:serum/s01_bottling' })

    // 使用相同 ID 会替换原配方。
    event.recipes.cybercultivator.serum_bottling(
        'cybercultivator:synaptic_serum_s01',
        ['minecraft:glass_bottle']
    ).id('cybercultivator:serum/s01_bottling')
})
```

不同 ID 的配方同时匹配时，固定按 `priority` 降序、配方 ID 升序选择。灌装机若已选中的最高优先级配方输出受阻，会等待输出槽腾空，不会降级执行另一条配方。Java 查询 API、机器和 JEI 使用相同排序。

### 可热重载玩法事件

四个专用事件均可放在 `server_scripts`，执行 `/reload` 后更新，无需重启游戏：

```javascript
CyberCultivatorEvents.geneSplice(event => {
    event.setSpeed(Math.min(10, event.getSpeed() + 1))
    event.setOffspringCount(2)
    if (event.getGeneration() > 20) event.cancel()
})

CyberCultivatorEvents.cropMature(event => {
    if (`${event.getOutput().id}` === 'cybercultivator:plant_fiber') {
        event.setOutput(Item.of('cybercultivator:plant_fiber', 2))
    }
})

CyberCultivatorEvents.serumCraft(event => {
    if (`${event.getRecipeId()}` === 'kubejs:fast_s01') {
        event.setActivity(12)
        event.setOutput('cybercultivator:synaptic_serum_s01')
    }
})

CyberCultivatorEvents.serumConsume(event => {
    if (event.getActivity() >= 10) {
        event.setDuration(event.getDuration() * 2)
        event.setAmplifier(event.getAmplifier() + 1)
    }
})
```

每个事件都支持 `event.cancel()`。`GeneSpliceEvent` 的 `offspringCount` 在监听器完成后被限制为 `1..2`，最终突变状态始终强制为 2。`CropMatureEvent` 和 `SerumCraftEvent` 的 `getOutput()` 返回防御性副本，修改产出必须调用 `setOutput()`。Activity 的有效范围为 `1..15`。

低级入口 `ForgeEvents.onEvent(...)` 仍可使用，但只能从 `startup_scripts` 注册，脚本修改后必须重启；新脚本应优先使用上述 `CyberCultivatorEvents`。

执行 `/reload` 时配方和专用事件监听器会一起重载。如果灌装机正在执行的配方对象被替换或删除，本次进度会清零且不会消耗输入；随后机器会按新配方重新匹配。

## 常见问题

**Q: 作物一直不生长？**
A: 打开培养槽 GUI 检查 N/P/D 数值。需要 N > 10 且 P > 10 且 D > 0 才能生长；潜行右键只用于取回种子。

**Q: 基因数值怎么看？**
A: 佩戴光谱单片镜后，将鼠标悬停在基因种子上即可查看 Gene_Speed / Gene_Yield / Gene_Potency 数值。未佩戴时显示「需要佩戴光谱单片镜以解析基因」。

**Q: 血清副作用太强怎么办？**
A: 佩戴生命支持箱可加速副作用消退。或在安全环境下使用血清，等待副作用自然结束。

**Q: 没有 Curios API 能用饰品吗？**
A: 不能。饰品功能依赖 Curios API。但血清系统、培养槽、拼接机等核心功能不受影响。

**Q: 为什么没有收到游戏内指导书？**
A: 指导书需安装 Patchouli `1.20.1-85-FORGE`。安装后重新登录即会发放；也可用原版书和硅碎片合成补领。不安装 Patchouli 不影响核心玩法。

**Q: 矿石不生成？**
A: 硅晶矿石和稀土矿石使用 Forge biome modifier 生成；石头层使用普通变种，负高度深板岩层使用深板岩变种。正常创建新世界或探索新生成区块即可找到。

## 升级注意事项

### 从 v1.1.x 升级到 v1.2.0

**配置文件迁移：** v1.2.0 将配置类型从 `COMMON` 改为 `SERVER`。旧版文件 `config/cybercultivator-common.toml` 不再被读取，新配置文件位于 `<存档>/serverconfig/cybercultivator-server.toml`。升级步骤：

1. 启动一次世界让模组生成默认 `cybercultivator-server.toml`
2. 将旧 `config/cybercultivator-common.toml` 中的自定义值手动复制到新文件
