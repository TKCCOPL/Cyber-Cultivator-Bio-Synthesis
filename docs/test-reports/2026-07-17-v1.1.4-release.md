# v1.1.4 发布自动化验证报告（2026-07-17）

## 范围

本次更新将版本提升至 `1.1.4`，为 CI 增加可下载 JAR artifact，并在版本 PR 合入 `main` 且全部验证通过后自动创建 annotated tag 和 GitHub Release。生产 Java 代码及游戏资源未改动。

CI 同时复用 Gradle User Home 与 build cache，将 `build`/`runData` 合并为一次 Gradle 启动，并跳过纯 Markdown/`docs/` 变更的重型 Forge 验证。Curios/JEI 完整依赖与无可选依赖两组运行时测试仍保留并行执行。

合并前复核还将 Forge datagen 的 `.cache/` 排除出资源，并关闭 JAR 文件时间戳、固定归档顺序，避免本地缓存或构建时间污染发布产物。

## 发布保护

- Release 已存在时安全跳过，仍保留 30 天的 workflow artifact。
- Tag 已存在但指向其他提交且没有 Release 时停止发布，要求提升 `mod_version`。
- 发布作业是唯一获得 `contents: write` 的作业，并等待构建和两组运行时测试成功。
