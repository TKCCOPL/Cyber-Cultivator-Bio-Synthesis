### 详细审查记录

#### 1. 核心交互路径

**BioIncubatorBlock + BioIncubatorBlockEntity:**
- 种子插入 -> 资源注入(water/purity/nutrition/signal) -> tick 增长 -> 成熟 -> 作物产出掉落
- 交互流: 右键检查种子插入, 资源注入(water bucket/biochemical solution/silicon shard), shift+右键取回
- tick 方法在 seed.isEmpty() 时提前返回，避免空指针
- getGene() 对空 NBT 返回默认值 1，防止生长计算异常
- 路径完整: PASS

**GeneSplicerBlock + GeneSplicerBlockEntity:**
- 插入种子 A -> 插入种子 B -> craftOutput(breedGene) -> output 就绪 -> 取出
- 右键放种子, shift+右键取出 output 或 input, 空手取出 output
- 防抖机制(lastInsertTick)防止同 tick 双次插入
- breedGene: 父本均值 + 变异(-2..+2), clampGene 保证 1-10
- 路径完整: PASS

**AtmosphericCondenserBlock + AtmosphericCondenserBlockEntity:**
- tick 产出纯净水 -> 自动传输 purity 到下方培养槽 -> 右键取出
- 600 tick 产出, 库存上限 32, 自动传输阈值 purity < 80
- WorldlyContainer 实现漏斗兼容
- 路径完整: PASS

**SerumBottlerBlock + SerumBottlerBlockEntity:**
- 插入 3 材料 -> tick 匹配配方 -> 加工 300 tick -> output 就绪 -> shift+右键取出
- 右键放入材料到第一个空槽, 空手先取 output 再取 input
- cancelProcessing() 在取回 input 时正确重置 activeRecipe/progress/maxProgress
- getRecipeOutput() 在 consumeInputs() 之前执行，确保读取到输入 NBT
- 路径完整: PASS

**SynapticSerumItem (血清饮用链路):**
- 饮用 -> 根据 Activity 缩放持续时间 -> applyEffect -> 效果结束 -> NeuralOverload
- finishUsingItem: amplifier 叠加(上限 4), 持续时间累加(上限 5 分钟)
- 副作用链: removeAttributeModifiers 检查 entity.getEffect(this) == null 防止叠加替换时重复施加
- 路径完整: PASS

#### 4. 合成配方

**Activity 公式 (SerumBottlerBlockEntity.calculateActivity):**
```
raw = potency * 0.25 + purity * 0.375 + concentration * 0.375
result = clamp(round(raw), 1, 10)
```
- 按物品种类查找输入，不依赖槽位顺序 -- 正确
- 无 NBT 时默认值 5 (平衡点) -- 正确

**基因拼接公式 (GeneSplicerBlockEntity.breedGene):**
```
value = (parentA + parentB) / 2 + random.nextInt(5) - 2
result = clampGene(value)  // 1-10
```
- 期望值 = (A+B)/2, 方差由随机变异贡献 -- 与设计文档一致

**血清效果缩放 (SynapticSerumItem):**
```
multiplier = 0.5 + activity * 0.1   // activity 1-10 -> multiplier 0.6-1.5
scaledDuration = round(baseDuration * multiplier)
baseAmplifier = activity >= 8 ? 1 : 0
```
- 叠加时: amp = min(existing.amp + 1, 4), duration = min(scaled + remaining, 6000) -- 正确

#### 5. NBT 边界

| 字段 | clamp 范围 | 读取保护 | 备注 |
|------|-----------|---------|------|
| nutrition | 0-100 | clampStat() | load 时 clamp |
| purity | 0-100 | clampStat() | load 时 clamp |
| dataSignal | 0-100 | clampStat() | load 时 clamp |
| growthProgress | 0+ | Math.max(0, ...) | 无上限，成熟时重置 |
| gene values (1-10) | 1-10 | clampGene() | getGene 对 null NBT 返回 1 |
| SynapticActivity (1-10) | 1-10 | Math.max(1, min(10, ...)) | 无 NBT 返回 5 |
| amplifier | 0-4 | MAX_AMPLIFIER = 4 | finishUsingItem 中 clamp |
| serum duration | 上限 5min | Math.min(..., 20*300) | 累加时 clamp |

**空 tag 处理:**
- GeneticSeedItem.getGene(): `tag == null || !tag.contains(key)` 返回 1 -- 安全
- SynapticSerumItem.getActivity(): `tag == null || !tag.contains(TAG_ACTIVITY)` 返回 5 -- 安全
- SerumBottlerBlockEntity.getActivity(): 同上 -- 安全
- 所有 BlockEntity.load(): `tag.contains(TAG) ? ItemStack.of(...) : ItemStack.EMPTY` -- 安全

### 问题列表

1. **[轻微] SerumBottlerBlockEntity.removeItem() 缺少 cancelProcessing() 调用**
   - 位置: SerumBottlerBlockEntity.java:305-320
   - 问题: 漏斗通过 removeItem() 从 input 槽抽取材料时，不会调用 cancelProcessing()，导致 activeRecipe/progress/maxProgress 状态残留
   - 实际影响: 低。canPlaceItemThroughFace() 限制了漏斗只能从 DOWN 方向提取 output，不能提取 input。但 removeItem() 是 WorldlyContainer 接口的公开方法，理论上可被其他 mod 调用
   - 建议: 在 removeItem() 的 input 分支中，如果 maxProgress > 0，调用 cancelProcessing()

2. **[轻微] SerumBottlerBlockEntity.syncToClient() 未做 isClientSide 检查**
   - 位置: SerumBottlerBlockEntity.java:276-281
   - 问题: BioIncubatorBlockEntity.syncToClient() 有 `if (level != null && !level.isClientSide)` 保护，但 SerumBottlerBlockEntity 只检查 `if (level != null)`
   - 实际影响: 无。syncToClient() 仅从服务端代码路径调用 (cancelProcessing 由 Block.use 触发，服务端执行)
   - 建议: 添加 isClientSide 检查以保持一致性

3. **[轻微] NeuralOverloadEffect 的 HUNGER amplifier 使用 `0 + amplifier` 表达式**
   - 位置: NeuralOverloadEffect.java:22
   - 问题: `0 + amplifier` 是冗余表达式，直接写 `amplifier` 即可。当 amplifier=4 时，HUNGER IV 每 0.5 秒消耗 1.0 饥饿值，副作用非常严厉
   - 实际影响: 无功能问题。副作用严厉是设计意图 (crash-down 机制)
   - 建议: 简化为 `amplifier`，或添加注释说明 HUNGER IV 的设计意图
