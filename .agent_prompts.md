# 可复用示例 Prompt（用于 `资深 Minecraft 模组系统架构师与游戏策划` agent）

说明：将下面任意一条完整复制并发送给该 agent。Agent 会返回遵循规定输出格式的 Markdown（包含【机制概述】【注册表清单】【NBT 结构表】【交互流转图】）。约束：目标版本 `1.20.1`，允许联网检索实现细节。

---

## 示例 1：多方块合成机（进度条 + 能量 + 传送带输入）

角色/上下文：我需要一个多方块合成机（类似工业模组的合成炉），从传送带自动输入物品，有电力消耗、进度条和输出缓存。

要求：
- 返回完整的四段 Markdown（【机制概述】【注册表清单】【NBT 结构表】【交互流转图】）。
- 在【注册表清单】列出所有需要新增的 registry 对象（Block, Item, BlockEntityType, Menu, RecipeSerializer 等）。
- 在【NBT 结构表】为每个 TileEntity 字段提供：字段名、类型、默认值、上下限、是否同步、用途说明。
- 提供示例 Java 字段声明与同步策略的伪代码片段（`BlockEntity` 字段、`tick` 逻辑、`load/save`、`getUpdatePacket`/`onDataPacket`）。
- 包含一个示例 `recipe` JSON 片段和一个简单的 GUI 布局说明。

限制：兼容 Minecraft/Forge `1.20.1`。

---

## 示例 2：可基因编辑的作物（Seed/Plant/交配机制）

高概念：玩家可以通过 `Synaptic Serum` 等药剂编辑种子基因，从而改变作物属性（产量、成熟时间、耐旱等）。支持交配生成新品系。

要求：
- 输出遵循四段模板；在【NBT 结构表】特别列出基因槽（如 `geneA`、`geneB`）的数据类型与取值域（集合或枚举）。
- 列出必要的 Item/Block/BlockEntity/Menu/Recipe 注册项。
- 给出种子与作物的成长阶段与触发事件（例如：生长 tick、施肥、基因突变概率）的伪代码或流程说明。
- 明确哪些字段需要在客户端可视化（例如：基因显示、进度条），并给出同步频率建议。

限制：兼容 Minecraft/Forge `1.20.1`，若涉及 Curios 或 Capability，请在答复中声明具体接口。

---

## 示例 3：Curios 联动饰品（增长加成饰品）

场景：设计一个可装备在 Curios 饰品栏的 `Growth Booster`，它可以提高附近作物的生长速度，但有冷却与能量/耐久消耗。

要求：
- 在回答中列出 Curios 注册与交互点（slot 类型、事件监听、Capability 或 Curios API 的调用伪代码）。
- 在【NBT 结构表】列出饰品的持久化字段（如 `cooldown`、`charges`、`boundPlayer`），并说明是否序列化到 ItemStack NBT。
- 提供服务端检测与范围效应应用的伪代码（效率与频率考虑），并说明如何避免性能问题（例如：分摊检测、区块加载检查）。

限制：兼容 Minecraft/Forge `1.20.1`，Curios API 版本请自行检索并引用实现要点。
